01 — Foundation
Brand Story
What Cassette is, who it's for, and the feeling it should always create.
The Product
Cassette is a private family video scrapbook app. It takes the chaos of an iPhone camera roll and turns it into something worth watching again — curated, beautiful, and yours alone. No algorithm. No audience. Just the moments that matter.
The Primary User
A mom with 800 videos on her phone she hasn't looked at since Christmas. She's uploading from the waiting room at dance class — one hand, 10 minutes, patchy WiFi. Cassette should feel like a gift, not a task.
The Feeling
Ordo ab chao — order from chaos. Cassette takes the raw, messy abundance of family life and creates something warm, cozy, and worth revisiting. Like finding an old shoebox of photos. Like the smell of a projector warming up. Like pressing play on something you almost forgot.
The Name
Cassette. Named for the cassette tape — the format of the home video era. Warm, analog, intimate. Deliberately pre-Instagram. The name carries the weight of recorded memory without carrying any existing brand baggage.

05 — Components
Button System
All buttons use Plus Jakarta Sans. Rounded pill shape throughout — no sharp corners. One primary action per screen. Destructive actions use Sienna, never Red.
Button variants
✓ Do
One Primary (amber) button per screen. It's the one thing you want them to do next.
✗ Don't
Don't stack two Primary buttons. Don't use red for destructive actions — use Sienna.
✓ Do
Minimum touch target 44×44pt on all interactive elements. She's using one thumb.
✗ Don't
Don't use hover states as the primary interaction pattern. This is a touch-first app.

08 — Components
Motion Principles
Motion should feel warm and unhurried — like film, not software. Nothing snaps. Everything eases. Playback transitions are the most important animations in the product.
Default Ease
cubic-bezier(0.4, 0, 0.2, 1) · 200ms
Used for all standard UI transitions — show/hide, state changes, hover.
Gentle Enter
cubic-bezier(0, 0, 0.2, 1) · 300ms
Screens, sheets, and cards appearing. Starts fast, decelerates warmly.
Playback Swipe
cubic-bezier(0.25, 0.46, 0.45, 0.94) · 380ms
Clip-to-clip transition in the Reels viewer. Should feel like turning a page, not swiping a feed.
Caption Fade
ease-in-out · 500ms · opacity 0→1
Text captions fade in gently over video. Never slide. Never pop.
Micro-interactions
ease · 120–150ms
Button press states, selection checkmarks, icon taps. Fast but never instant.
09 — Voice
Tone & Voice
Cassette speaks like a thoughtful friend — warm, a little poetic, never technical. It never calls attention to itself. The product's job is to get out of the way and let the memories do the talking.
Onboarding — first time
Let's make something worth watching. Start by choosing some videos from your camera roll. Don't overthink it — you can always add more later.
Empty state — no scrapbooks yet
Your first scrapbook is waiting. Grab a batch of videos and let's turn them into something.
Upload progress
Gathering your videos… almost there.
Scrapbook complete
It's ready. Tap to watch.
Error — upload failed
Something didn't upload right. Try again when you have a better connection — your selections are saved.
✓ Voice
Warm. Quiet. A little poetic. Uses short sentences. Occasionally uses italics for emotional emphasis.
✗ Never
Technical jargon, exclamation points, emoji in UI copy, "upload successful!", corporate tone.
10 — Voice
Usage Rules
What to always do, what to never do. These rules protect the brand from drift as the product grows.
✓ Always
Dark background (Walnut) as default. Light mode is not a Cassette experience.
✗ Never
Never use the mark in any color other than Amber on Walnut. No white version, no dark version.
✓ Always
Mobile-first layouts. Design the iPhone experience first, adapt to desktop second.
✗ Never
Never use Inter, Roboto, Arial, or system fonts. Fraunces and Jakarta Sans, always.
✓ Always
Italic Fraunces for emotional moments — captions, the logo, hero text. It carries the warmth.
✗ Never
Never rotate, stretch, recolor, or add drop shadows to the Spool mark.
✓ Always
44px minimum touch targets. One Primary button per screen. Generous padding.
✗ Never
Never use red for destructive actions. Use Sienna (#E8855A) — it's warm, not alarming.
11 — Reference
CSS Tokens
Copy these into your global CSS or Tailwind config. Every color in the app should reference a token, never a raw hex value in component code.
/* Cassette — CSS Custom Properties */
:root {
  --cassette-amber: #F2A24A; /* Primary accent */
  --cassette-sienna: #E8855A; /* Secondary / danger */
  --cassette-wheat: #F5DEB3; /* Primary text */
  --cassette-walnut: #2C1A0E; /* App background */
  --cassette-walnut-mid: #3D2410; /* Cards / surfaces */
  --cassette-walnut-light: #4A2E18; /* Borders / dividers */
  --cassette-rust: #7A3B1E; /* Labels / muted text */
  --cassette-deep: #1A0F08; /* Playback BG */
  /* Typography */
  --cassette-font-display: 'Fraunces', serif;
  --cassette-font-ui: 'Plus Jakarta Sans', sans-serif;
}
